02

Prerequisites

Apple Silicon vs Intel

Homebrew installs to different locations depending on your chip. Check which you have:

Terminal 
uname -m
Apple Silicon (M1/M2/M3/M4)Intel
Homebrew prefix/opt/homebrew/usr/local
Result of uname -marm64x86_64

Where paths differ, this guide shows both. Use the tab toggle at the top of each step to switch.

Fix Git SSH Override (if applicable)

If Git is configured to rewrite HTTPS URLs to SSH, Homebrew's install and update commands will fail. Check for overrides first:

Terminal 
git config --global --get-all url."git@github.com:".insteadOf
git config --global --get-all url."ssh://git@github.com/".insteadOf

If either command returns output, remove the overrides:

Terminal 
git config --global --unset-all url."git@github.com:".insteadOf
git config --global --unset-all url."ssh://git@github.com/".insteadOf
03

Step-by-Step Installation

Install Homebrew

Homebrew is the package manager for macOS. Skip this step if you already have it.

Terminal 
/bin/bash -c "$(curl -fsSL --tlsv1.2 https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
Apple Silicon only After installation, add Homebrew to your PATH for the current session: eval "$(/opt/homebrew/bin/brew shellenv)"
Update Homebrew

Ensures your formula index is up to date before installing anything.

Terminal 
brew update --force
Install Visual Studio Code
Terminal 
brew install --cask visual-studio-code
Install Git
Terminal 
brew install git
Install GitHub CLI
Terminal 
brew install gh
Install wget
Terminal 
brew install wget
Install Python 3.12
Terminal 
brew install python@3.12
Install Node.js LTS

Node.js 22 is "keg-only" ‚Äî not automatically linked into your PATH. You'll configure this permanently in step 4.4, but for the current session run the appropriate command now:

Apple Silicon
brew install node@22
export PATH="/opt/homebrew/opt/node@22/bin:$PATH"
‚óª Intel
brew install node@22
export PATH="/usr/local/opt/node@22/bin:$PATH"

Do not use sudo If you get permission errors, fix npm's directory instead ‚Äî see the Troubleshooting section.

Uninstallation

Order matters Uninstall Claude Code first while npm is still available, then remove the other packages.
Uninstall Claude Code first
Terminal 
npm uninstall -g @anthropic-ai/claude-code
Uninstall recommended extras
Terminal 
brew uninstall jq ripgrep fzf httpie tree
Uninstall optional packages
Terminal 
brew uninstall --cask imageoptim
brew uninstall fd
Uninstall core packages
Terminal 
brew uninstall --cask visual-studio-code
brew uninstall node@22 python@3.12 gh wget git
Clean up shell configuration

Open your ~/.zshrc (or ~/.bash_profile) and remove these blocks:

Remove from ~/.zshrc 
# Homebrew
eval "$(/opt/homebrew/bin/brew shellenv)"

# Python 3.12
export PATH="/opt/homebrew/opt/python@3.12/libexec/bin:$PATH"

# Node.js
export PATH="/opt/homebrew/opt/node@22/bin:$PATH"
Remove Git configuration
Terminal 
git config --global --unset init.defaultBranch
git config --global --unset core.editor

Troubleshooting

"brew: command not found"
Homebrew is not installed or not in your PATH. Install it via step 3.1. On Apple Silicon, make sure you've added the brew shellenv line to your RC file (step 4.2).
"brew update" fails with "Permission denied (publickey)"
Git is configured to rewrite HTTPS URLs to SSH for GitHub. Follow the Fix Git SSH Override steps in section 2 to remove those overrides.
Homebrew doesn't recognize your macOS version
If brew config shows "unsupported" or "unknown" for your macOS version, reinstall Homebrew using the same command from step 3.1, then run brew update --force.
Node.js / npm commands not found after installation
Node.js 22 is "keg-only" and not auto-linked. Make sure your PATH includes the Node.js bin directory permanently via step 4.4, and reload your shell with source ~/.zshrc.
Python command not found
Homebrew's Python 3.12 is keg-only. Follow step 4.3 to add it to your PATH. You can also try python3 --version ‚Äî Homebrew Python may only be available as python3 before the PATH is configured.
npm install fails for Claude Code
Do not use sudo npm install. Instead, fix npm's global directory permissions:
Terminal 
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH="$HOME/.npm-global/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
"code" command not found after installing VS Code
Open VS Code manually, press Cmd+Shift+P, type "shell command", and select Shell Command: Install 'code' command in PATH.
