02

Prerequisites

System Requirements

Windows 10 or later (Windows 11 recommended). You must have Administrator privileges ‚Äî right-click PowerShell and choose Run as Administrator for all steps.

Ensure winget is available

winget (Windows Package Manager) comes pre-installed on Windows 11. On Windows 10, install App Installer from the Microsoft Store if the command below fails.

PowerShell (Admin) 
winget --version

If a version number is returned, you're ready to go.

Update winget sources
PowerShell (Admin) 
winget source update --disable-interactivity
03

Step-by-Step Installation

Open PowerShell as Administrator All commands below require elevated privileges. Each winget command uses --exact --id to get the correct package and --scope machine for a system-wide install.
Install Visual Studio Code
PowerShell (Admin) 
winget install --exact --id Microsoft.VisualStudioCode --source winget --scope machine --accept-source-agreements --accept-package-agreements --silent --disable-interactivity
Install Git for Windows
PowerShell (Admin) 
winget install --exact --id Git.Git --source winget --scope machine --accept-source-agreements --accept-package-agreements --silent --disable-interactivity
Install GitHub CLI
PowerShell (Admin) 
winget install --exact --id GitHub.cli --source winget --scope machine --accept-source-agreements --accept-package-agreements --silent --disable-interactivity
Install Python 3.12
PowerShell (Admin) 
winget install --exact --id Python.Python.3.12 --source winget --scope machine --accept-source-agreements --accept-package-agreements --silent --disable-interactivity
Install Node.js LTS
PowerShell (Admin) 
winget install --exact --id OpenJS.NodeJS.LTS --source winget --scope machine --accept-source-agreements --accept-package-agreements --silent --disable-interactivity
04

Environment Configuration

Restart PowerShell after PATH changes Close and reopen PowerShell as Administrator, or run the session-refresh commands in step 4.3 to apply changes immediately.
Add tools to System PATH

The installers usually handle this automatically. If any command isn't found, add its path manually:

PowerShell (Admin) 
# VS Code
[Environment]::SetEnvironmentVariable("Path", [Environment]::GetEnvironmentVariable("Path", "Machine") + ";C:\Program Files\Microsoft VS Code\bin", "Machine")

# Git
[Environment]::SetEnvironmentVariable("Path", [Environment]::GetEnvironmentVariable("Path", "Machine") + ";C:\Program Files\Git\cmd", "Machine")

# Python 3.12 + Scripts
[Environment]::SetEnvironmentVariable("Path", [Environment]::GetEnvironmentVariable("Path", "Machine") + ";C:\Program Files\Python312;C:\Program Files\Python312\Scripts", "Machine")

# Node.js
[Environment]::SetEnvironmentVariable("Path", [Environment]::GetEnvironmentVariable("Path", "Machine") + ";C:\Program Files\nodejs", "Machine")
Tip: Check before adding Verify a path isn't already present to avoid duplicates: [Environment]::GetEnvironmentVariable("Path", "Machine") -split ";" | Select-String "VS Code"
Set PYTHONUTF8 environment variable

Forces Python to default to UTF-8 encoding, preventing character encoding issues.

PowerShell (Admin) 
[Environment]::SetEnvironmentVariable("PYTHONUTF8", "1", "Machine")
Refresh current session

Apply the new PATH and variables without restarting PowerShell:

PowerShell (Admin) 
$env:Path = [Environment]::GetEnvironmentVariable("Path", "Machine") + ";" + [Environment]::GetEnvironmentVariable("Path", "User")
$env:PYTHONUTF8 = "1"
Configure Git

Set at the system level (applies to all users) since we're running as Administrator:

PowerShell (Admin) 
git config --system core.autocrlf true
git config --system init.defaultBranch main
Upgrade pip
PowerShell (Admin) 
python -m pip install --upgrade pip --quiet

Uninstallation

Order matters Uninstall Claude Code first while npm is still available, then remove the other packages.
Uninstall Claude Code first
PowerShell (Admin) 
npm uninstall -g @anthropic-ai/claude-code
Uninstall optional packages
PowerShell (Admin) 
winget uninstall --exact --id voidtools.Everything.Cli --source winget --silent --accept-source-agreements --disable-interactivity
winget uninstall --exact --id voidtools.Everything     --source winget --silent --accept-source-agreements --disable-interactivity
Uninstall core packages
PowerShell (Admin) 
winget uninstall --exact --id Microsoft.VisualStudioCode --source winget --silent --accept-source-agreements --disable-interactivity
winget uninstall --exact --id Git.Git                    --source winget --silent --accept-source-agreements --disable-interactivity
winget uninstall --exact --id GitHub.cli                 --source winget --silent --accept-source-agreements --disable-interactivity
winget uninstall --exact --id Python.Python.3.12         --source winget --silent --accept-source-agreements --disable-interactivity
winget uninstall --exact --id OpenJS.NodeJS.LTS           --source winget --silent --accept-source-agreements --disable-interactivity
Clean up PATH entries
PowerShell (Admin) 
$currentPath = [Environment]::GetEnvironmentVariable("Path", "Machine")
$patterns    = @("*\Microsoft VS Code\*", "*\Git\*", "*\Python312*", "*\nodejs*", "*\npm*")
$pathArray   = $currentPath -split ";" | Where-Object { $_ -ne "" }

foreach ($pattern in $patterns) {
    $pathArray = @($pathArray | Where-Object { $_ -notlike $pattern })
}

[Environment]::SetEnvironmentVariable("Path", ($pathArray -join ";"), "Machine")
Remove environment variables & Git config
PowerShell (Admin) 
[Environment]::SetEnvironmentVariable("PYTHONUTF8", $null, "Machine")
git config --system --unset core.autocrlf
git config --system --unset init.defaultBranch
09

Troubleshooting

"winget is not recognized"
On Windows 10, install App Installer from the Microsoft Store. On Windows 11, it should be pre-installed ‚Äî try opening a fresh PowerShell window and running winget again.
"Access denied" or "requires Administrator"
Right-click on PowerShell and select Run as Administrator. System-level PATH changes and git config --system require elevated privileges.
Package "already installed" messages
This is expected and safe to ignore. winget simply skips packages that are already present. Exit code -1978335189 means "already installed."
Commands not found after installation
Close all terminal windows and open a fresh one. PATH changes only take effect in new sessions. Alternatively, run the session-refresh commands from step 4.3 to update the current session immediately.
npm install fails for Claude Code
Verify Node.js is installed: npm --version. If you get permission errors, ensure you're in an Administrator PowerShell. You can also try clearing the npm cache first: npm cache clean --force
Python encoding issues
Verify the variable is set: [Environment]::GetEnvironmentVariable("PYTHONUTF8", "Machine") ‚Äî it should return 1. If it does and Python still misbehaves, restart your terminal to pick up the change.
Git line ending issues
Verify the setting is applied: git config --system core.autocrlf ‚Äî should return true. This ensures consistent Windows line endings across all repositories.
