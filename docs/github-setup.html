01

Create a GitHub Account

GitHub is where your code lives on the internet. It's free, and you'll use it to back up your projects and collaborate with others (including AI assistants).

Go to GitHub

Open your browser and go to github.com. Click Sign up in the top-right corner.

Choose a username

Pick something professional-ish ‚Äî this will be part of your profile URL (e.g., github.com/your-username). You can always change it later, but it's easier to pick a good one now.

Username tips Keep it short, lowercase, and easy to type. Avoid special characters. Something like your name or a consistent handle works great.
Enter your email and password

Use a real email you check ‚Äî GitHub will send you a verification code. Choose a strong password.

Verify your email

Check your inbox for a verification email from GitHub. Click the link or enter the code they sent. Once verified, your account is ready.

Configure your Git identity

Tell Git who you are ‚Äî this info shows up on your commits. Replace the name and email with your own:

Terminal 
git config --global user.name "Your Name"
git config --global user.email "your-email@example.com"
Use the same email Use the same email you signed up with on GitHub. This links your local commits to your GitHub profile.
02

Set Up SSH Keys

SSH keys let your computer prove its identity to GitHub without typing your password every time. Think of it as a digital handshake ‚Äî your computer has a private key, and GitHub has the matching public key.

Check for existing SSH keys

You might already have one. Let's check first:

Terminal 
ls -la ~/.ssh

If you see files like id_ed25519 and id_ed25519.pub, you already have a key pair ‚Äî skip to step 2.3. If the folder doesn't exist or is empty, continue to the next step.

Generate a new SSH key pair

Replace the email with the one you used for GitHub:

Terminal 
ssh-keygen -t ed25519 -C "your-email@example.com"

It will ask you a few things:

What to type "Enter file in which to save the key" ‚Äî press Enter to accept the default location.
"Enter passphrase" ‚Äî you can press Enter for no passphrase (simpler), or type one for extra security.
Add your key to the ssh-agent

Start the SSH agent and add your key so it's ready to use:

Terminal (macOS) 
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
On macOS You can add --apple-use-keychain to the ssh-add command to store the passphrase in your macOS Keychain so you don't have to type it again.
PowerShell (Windows) 
# Start the ssh-agent service
Get-Service ssh-agent | Set-Service -StartupType Automatic
Start-Service ssh-agent
ssh-add $env:USERPROFILE\.ssh\id_ed25519
Copy your public key

Copy the public key to your clipboard. This is the key you'll give to GitHub ‚Äî never share your private key (the file without .pub).

Terminal (macOS) 
pbcopy < ~/.ssh/id_ed25519.pub
PowerShell (Windows) 
Get-Content $env:USERPROFILE\.ssh\id_ed25519.pub | Set-Clipboard

The key is now in your clipboard ‚Äî it starts with ssh-ed25519.

Add the key to GitHub

Now paste it into your GitHub account:

Step by step 1. Go to github.com and click your profile picture (top-right) ‚Üí Settings
2. In the left sidebar, click SSH and GPG keys
3. Click New SSH key
4. Give it a title (e.g., "My Laptop") and paste the key into the "Key" field
5. Click Add SSH key
Test the connection

Verify everything works:

Terminal 
ssh -T git@github.com

If it asks "Are you sure you want to continue connecting?" type yes and hit Enter.

Success looks like Hi your-username! You've successfully authenticated, but GitHub does not provide shell access. ‚Äî this means it worked!
03

Connect VS Code to GitHub

VS Code has built-in Git support and a GitHub extension. Let's connect them so you can manage your code right from the editor.

Open VS Code

Launch VS Code. If you installed it with the Vibe Coding Stack script, you can open it from the terminal:

Terminal 
code
Sign in to GitHub from VS Code

VS Code may prompt you to sign in automatically. If not:

How to sign in 1. Click the Accounts icon in the bottom-left of the sidebar (person icon)
2. Click Sign in with GitHub
3. Your browser opens ‚Äî authorize VS Code to access your GitHub account
4. You'll be redirected back to VS Code. Done!
Install the GitHub Pull Requests extension (optional)

For a richer experience, install the official GitHub extension:

Terminal 
code --install-extension GitHub.vscode-pull-request-github
04

Your First Repo

A "repo" (repository) is just a project folder that Git tracks. Let's create one on GitHub and bring it to your computer.

Create a repo on GitHub
Step by step 1. Go to github.com and click the + button (top-right) ‚Üí New repository
2. Name it something simple like my-first-project
3. Choose Public or Private (either is fine)
4. Check "Add a README file"
5. Click Create repository
Clone it to your computer

On your repo page, click the green Code button, select the SSH tab, and copy the URL. Then clone it:

Terminal 
git clone git@github.com:your-username/my-first-project.git

This creates a folder called my-first-project with your repo inside it.

Open it in VS Code
Terminal 
code my-first-project

VS Code opens with your project. You'll see the README.md file in the file explorer on the left.

05

Commit & Push

Now let's make a change, save it with Git (commit), and send it to GitHub (push). This is the core workflow you'll use every day.

Make a change

Open README.md in VS Code and add a line ‚Äî anything you want. For example:

README.md 
# my-first-project

Hello world! This is my first change.

Save the file (Ctrl+S on Windows, ‚åò+S on Mac).

Stage, commit, and push ‚Äî VS Code method

The easiest way is right inside VS Code:

Using the VS Code sidebar 1. Click the Source Control icon in the left sidebar (looks like a branch)
2. You'll see your changed file listed. Click the + next to it to stage it
3. Type a message in the text box at the top, like Add greeting to README
4. Click the Commit button (checkmark)
5. Click the Sync Changes button (or the cloud icon) to push to GitHub
Stage, commit, and push ‚Äî Terminal method

You can also do everything from the terminal. Open the terminal in VS Code (Ctrl+` or ‚åò+`):

Terminal (inside VS Code) 
# Stage the changed file
git add README.md

# Commit with a message
git commit -m "Add greeting to README"

# Push to GitHub
git push
Verify on GitHub

Go to your repo on github.com and refresh the page. You should see your changes in the README! Your code is now backed up on the internet.

You did it! You've just completed the core Git workflow: change ‚Üí stage ‚Üí commit ‚Üí push. This is the same process developers use every day, whether they're working alone or on a team of hundreds.
